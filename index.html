< article>: El elemento Contenido del artículo El elemento HTML < article> representa una composición independiente en un documento, página, aplicación o sitio, que está destinado a ser distribuible o reutilizable de forma independiente . Cada < artícule> debe identificarse, normalmente mediante la inclusión de un encabezado (elemento < h1> - < h6>) como elemento secundario del elemento < artículo>. Cuando se anida un elemento < article>, el elemento interior representa un artículo relacionado con el elemento exterior. Por ejemplo, los comentarios de una publicación de blog pueden ser elementos < article> anidados en el < article> que representa la publicación de blog. La información del autor de un elemento < article> se puede proporcionar a través del elemento < address>, pero no se aplica a los elementos < article> anidados. La fecha y la hora de publicación de un elemento < article> se pueden describir utilizando el atributo datetime de un elemento < time>. algunos ejemplos de < article> son Publicación en un foro, artículos de revistas o periódicos, una entrada de blog, una ficha de producto, etc.
< aside>: El elemento Aside El elemento HTML < aside> representa una parte de un documento cuyo contenido solo está indirectamente relacionado con el contenido principal del documento. No utilice el elemento < aside> para etiquetar texto entre paréntesis, ya que este tipo de texto se considera parte del flujo principal. Los asides se presentan con frecuencia como barras laterales o cuadros de llamada
< body>: el elemento del cuerpo del documento El elemento HTML < body> representa el contenido de un documento HTML. Solo puede haber un elemento < body> en un documento EL elemento < body> un ejemplo seria qualquier pagina principal de un sitio web aun que explicitamente no se ve
< div>: el elemento de división de contenido El elemento HTML < div> es el contenedor genérico para el contenido de flujo. El elemento < div> debe usarse solo cuando ningún otro elemento semántico (como < article> o < nav>) sea apropiado. El contenido del elemento < div> puede ser qualquiera solo cambia utilizando CSS
< footer> El elemento HTML < footer> representa un pie de página para su ancestro más cercano que secciona el contenido o secciona el elemento raíz. Información privada sobre el autor en un elemento < address> que se pueda incluir en el elemento < footer>. Cuando el ancestro más cercano que secciona el contenido o el elemento raíz que secciona es el elemento del cuerpo, el pie de página se aplica a toda la página. El elemento < footer> no secciona el contenido y, por lo tanto, no introduce una nueva sección en el esquema. Un < footer> normalmente contiene información sobre el autor de la sección, datos de derechos de autor o enlaces a documentos relacionados.
< h1>–< h6>: Los elementos de encabezado de sección HTML Los elementos HTML < h1> a < h6> representan seis niveles de encabezados de sección. < h1> es el nivel de sección más alto y < h6> es el más bajo. Las aplicaciones de usuario pueden utilizar la información de encabezado para construir automáticamente una tabla de contenido para un documento. No utilice elementos de encabezado para cambiar el tamaño del texto. En su lugar, utilice la propiedad CSS font-size. No omita los niveles de encabezado: comience siempre desde < h1>, seguido de < h2> y así sucesivamente. Normalemnte se utiliza para destacar titulos.
< head>: el elemento de metadatos del documento (encabezado) El elemento HTML < head> contiene información legible por máquina (metadatos) sobre el documento. Se puede utilizar para títulos, scripts y hojas de estilo.
< header> El elemento HTML < header> representa contenido introductorio, normalmente un grupo de ayudas introductorias o de navegación. El elemento < header> tiene un significado idéntico al de la función de punto de referencia del banner de todo el sitio, a menos que esté anidado dentro del contenido de la sección. Entonces, el elemento < header> no es un hito. El elemento < header> puede definir un encabezado de sitio global, descrito como un banner en el árbol de accesibilidad. Por lo general, incluye un logotipo, el nombre de la empresa, una función de búsqueda y, posiblemente, la navegación global o un eslogan. Generalmente se encuentra en la parte superior de la página.De lo contrario, es una sección en el árbol de accesibilidad y generalmente contiene el encabezado de la sección circundante (un elemento h1 - h6) y un subtítulo opcional, pero esto no es obligatorio. Puede contener algunos elementos de encabezado, pero también un logotipo, un formulario de búsqueda, el nombre del autor y otros elementos.
< html>: el documento HTML/elemento raíz El elemento HTML < html> representa la raíz (elemento de nivel superior) de un documento HTML, por lo que también se denomina elemento raíz. Todos los demás elementos deben ser descendientes de este elemento.
< main> El elemento HTML < main> representa el contenido dominante del < body> de un documento. El área de contenido principal consiste en contenido que está directamente relacionado o amplía el tema central de un documento o la funcionalidad central de una aplicación Un documento no debe tener más de un elemento < main> que no tenga especificado el atributo oculto.
< meta>: el elemento de metadatos El elemento HTML < meta> representa metadatos que no pueden ser representados por otros elementos meta-relacionados de HTML Algunos ejemplos como base, link, script, style o title
< p>: El elemento Párrafo El elemento HTML < p> representa un párrafo. Los párrafos generalmente se representan en los medios visuales como bloques de texto separados de los bloques adyacentes por líneas en blanco y/o sangría en la primera línea. los párrafos HTML pueden ser cualquier agrupación estructural de contenido relacionado. Algunos ejemplos pueden ser como imagenes o campos de formulario.
< sección>: El elemento Sección Genérica El elemento HTML < section> representa una sección independiente genérica de un documento. solo debe usarse si no hay un elemento más específico para representarlo. Por ejemplo, un menú de navegación debería incluirse en un elemento < nav>, pero una lista de resultados de búsqueda o una visualización de mapa y sus controles no tienen elementos específicos y podrían colocarse dentro de una < sección>. < span>: el elemento de extensión de contenido El elemento HTML < span> es un contenedor genérico en línea para redactar contenido, que no representa inherentemente nada. Se puede usar para agrupar elementos con fines de estilo (usando los atributos class o id), o porque comparten valores de atributo, como lang. Debe usarse solo cuando ningún otro elemento semántico sea apropiado. < span> es muy parecido a un elemento < div>, pero < div> es un elemento a nivel de bloque mientras que < span> es un elemento en línea.
< strong>: El elemento de gran importancia El elemento HTML < strong> indica que su contenido tiene una gran importancia, seriedad o urgencia. El elemento < strong> es para contenido que es de "gran importancia", incluidas cosas de gran seriedad o urgencia (como advertencias). Esta podría ser una oración que sea de gran importancia para toda la página, o simplemente podría tratar de señalar que algunas palabras son de mayor importancia en comparación con el contenido cercano. Normalmente, este elemento se representa de forma predeterminada con un peso de fuente en negrita. Sin embargo, no debe usarse para aplicar un estilo audaz; use la propiedad CSS font-weight para ese propósito. Utilice el elemento < b> para llamar la atención sobre un texto determinado sin indicar un mayor nivel de importancia. Use el elemento < em> para marcar el texto que tiene énfasis. Otro uso aceptado para < strong> es para indicar las etiquetas de los párrafos que representan notas o advertencias dentro del texto de una página. Los navegadores normalmente muestran el contenido en negrita.
style: el elemento de información de estilo El elemento HTML style contiene información de estilo para un documento o parte de un documento. Contiene CSS, que se aplica al contenido del documento que contiene el elemento style
< tabla>: El elemento Tabla El elemento HTML < table> representa datos tabulares, es decir, información presentada en una tabla bidimensional compuesta por filas y columnas de celdas que contienen datos.
< tbody>: el elemento del cuerpo de la tabla El elemento HTML < tbody> encapsula un conjunto de filas de tabla (elementos < tr>), lo que indica que comprenden el cuerpo de la tabla (< table>). El elemento < tbody>, junto con sus elementos < thead> y < tfoot> relacionados, proporcionan información semántica útil que se puede utilizar al renderizar para pantalla o impresora.
< td>: El elemento Celda de datos de la tabla El elemento HTML < td> define una celda de una tabla que contiene datos. Participa en el modelo de tabla. Se puede utilizar para hacer tablas.
< tfoot>: El elemento Pie de tabla El elemento HTML < tfoot> define un conjunto de filas que resumen las columnas de la tabla. Se puede utilizar para hacer tablas.
< th>: el elemento de encabezado de tabla El elemento HTML < th> define una celda como encabezado de un grupo de celdas de tabla. La naturaleza exacta de este grupo está definida por los atributos de alcance y encabezados. Se puede utilizar para hacer tablas.
< thead>: el elemento de encabezado de tabla El elemento HTML < thead> define un conjunto de filas que definen el encabezado de las columnas de la tabla. Se puede utilizar para hacer tablas
< título>: el elemento Título del documento El elemento HTML < title> define el título del documento que se muestra en la barra de título del navegador o en la pestaña de una página. Solo contiene texto; las etiquetas dentro del elemento se ignoran.
< tr>: El elemento Fila de la tabla El elemento HTML < tr> define una fila de celdas en una tabla Las celdas de la fila se pueden establecer usando una combinación de elementos < td> (celda de datos) y < th> (celda de encabezado)
< ul>: El elemento Lista desordenada El elemento HTML < ul> representa una lista desordenada de elementos, normalmente representada como una lista con viñetas El elemento < ul> es para agrupar una colección de elementos que no tienen un orden numérico, y su orden en la lista no tiene sentido.
